❯ How to delete all the data and start fresh all over

● Two options depending on how thorough you want to be:

  ---
  Option A — Data only (keep containers running)

  Delete all MBD indices, clear the log, re-run:

  # 1. Delete every mbd-misbehaviors-* index
  curl -s "http://localhost:9200/_cat/indices?h=index&index=mbd-misbehaviors*" \
    | xargs -r -I{} curl -s -o /dev/null -w "Deleted {}: %{http_code}\n" \
        -X DELETE "http://localhost:9200/{}"

  # 2. Run fresh (truncates log, restarts Logstash, rebuilds alias)
  make fresh DATA=data/your-file.zip

  ---
  Option B — Full reset (wipe everything including Kibana/ES state)

  # 1. Tear down containers and remove all Docker volumes
  docker compose down -v

  # 2. Bring everything back up
  docker compose up -d

  # 3. Wait ~30 s for ES + Kibana to be ready, then re-run setup
  #    (re-imports dashboards, index template, alias)
  bash elk/setup.sh

  # 4. Detect and ingest fresh data
  make fresh DATA=data/your-file.zip

  ---
  Use Option A if you just want clean data but keep Kibana dashboards intact. Use Option B if something is broken at the
  infrastructure level and you want a completely clean slate.

 Would it be easy to add  sliderd to dynamically adjust the threshlds?

● Searched for 1 pattern, read 1 file (ctrl+o to expand)

● It's already there — --setup-kibana adds exactly that. But there's an important distinction between two kinds of threshold
   control.

  ---
  What --setup-kibana already gives you

  python manage_display_filter.py --es-url http://localhost:9200 \
                                   --kibana-url http://localhost:5601 \
                                   --setup-kibana

  This patches the dashboard with a Kibana Controls panel containing range sliders for every numeric field —
  heading_rate_deg_s, diff_kmh, yaw_diff_deg_s, etc. Moving a slider immediately narrows what the dashboard shows, no
  restart required.

  ---
  The two-tier picture

  Raw data (mbd-misbehaviors-*)
         │
         ▼
    ES alias filter  ←── thresholds.json  ←── manage_display_filter.py
    (mbd-display)         (static floor)        (run to change floor)
         │
         ▼
    Kibana Controls sliders  ←── user drags in browser
    (interactive, on top)         (instant, no restart)
         │
         ▼
    Dashboard panels

  - Alias filter = the permanent server-side floor. Records that don't meet it never reach Kibana at all. Changing it
  requires editing thresholds.json and re-running the script.
  - Controls sliders = an additional client-side filter on top of whatever the alias exposes. Instant, interactive,
  per-session.

  ---
  Practical workflow

  If you want to use sliders to discover good threshold values:

  1. Temporarily lower (or remove) the L2 alias thresholds so all L1 data is visible in the mbd-display view
  2. Use the sliders to explore the data and find where the clean/dirty boundary sits
  3. Copy those values into thresholds.json and run manage_display_filter.py to make them permanent

  If the sliders are just for ad-hoc exploration on top of the already-filtered set, run --setup-kibana once and they'll be
  there.
